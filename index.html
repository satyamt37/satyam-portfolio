Code Showcase

Apex Engineering Lab

Production-style service, trigger handler, and resilient callout orchestration pattern used for enterprise B2B flows.

Live Pattern

Queueable + Service Layer + Guarded Trigger

This pattern keeps trigger logic slim, routes heavy work to async processing, and centralizes business rules in a testable service class.

  • Bulk-safe from the first line of trigger execution.
  • Idempotent queue handling with payload snapshots.
  • Explicit retry hooks for external integration failures.
  • Clean separation across selector, service, and integration layers.
OrderSyncService.cls 
public with sharing class OrderSyncService {
  // Entrypoint used by trigger handlers
  public static void enqueueSync(Set<Id> orderIds){
    if(orderIds == null || orderIds.isEmpty()) return;
    System.enqueueJob(new OrderSyncQueueable(new List<Id>(orderIds)));
  }

  public class OrderSyncQueueable implements Queueable, Database.AllowsCallouts {
    private final List<Id> queuedOrderIds;
    public OrderSyncQueueable(List<Id> orderIds){
      queuedOrderIds = orderIds;
    }

    public void execute(QueueableContext context){
      List<Order> ordersToSync = [
        SELECT Id, OrderNumber, Status, EffectiveDate, TotalAmount
        FROM Order WHERE Id IN :queuedOrderIds
      ];
      OrderPayloadBuilder.Payload payload = OrderPayloadBuilder.fromOrders(ordersToSync);
      try {
        ExternalOrderClient.upsertOrders(payload);
      } catch(Exception ex){
        SyncRetryService.registerFailure(queuedOrderIds, ex.getMessage());
      }
    }
  }
}
